# 绿界支付审核页面访问指南

## 📋 审核期间配置

为了让绿界科技审核人员可以查看储值页面进行审核，需要确保：

### 1. 需要登录访问

⚠️ **注意**：储值页面需要登录后才能访问。审核人员需要使用测试账号登录后查看页面。

### 2. 显示审核中状态

**环境变量配置：**
```env
ECPAY_ENABLED=false
ECPAY_STATUS=pending
```

**效果：**
- 页面正常显示所有 UI 元素
- 显示"储值功能审核中"的提示卡片
- 储值方案卡片被隐藏（替代为审核中提示）
- 所有功能说明、安全保障等信息正常显示

### 3. 提供给绿界审核的访问方式

**方式 1：提供测试账号**
- 创建一个测试账号供审核人员使用
- 提供登录链接：`https://predictiongod.app/login`
- 登录后访问：`https://predictiongod.app/topup`

**方式 2：提供审核专用账号**
- 为审核人员创建专用账号
- 账号信息单独提供给绿界审核团队

**储值页面链接（需要登录后访问）：**
```
https://predictiongod.app/topup
```

**页面内容包括：**
- ✅ 页面标题和说明
- ✅ 储值方案展示区域（显示审核中提示）
- ✅ 支付方式说明（信用卡 / Apple Pay）
- ✅ 安全保障说明（SSL 加密）
- ✅ 即時到帳说明
- ✅ 溫馨提示
- ✅ 賣家資訊（绿界验证信息）

## 🎨 审核期间页面展示

### 正常显示的内容
- 页面头部（标题、返回按钮）
- 余额显示区域（未登录时显示欢迎信息）
- 功能说明卡片（支付方式、安全保障、即时到帐）
- 儲值方案区域（显示"审核中"提示）
- 溫馨提示
- 賣家資訊（包含绿界验证通过标识）

### 隐藏/禁用的功能
- 储值方案卡片（被"审核中"提示替代）
- 创建订单功能（后端返回 503）
- 余额显示（未登录时）

## 📝 审核要点

审核人员可以看到：

1. **完整的 UI 设计** - 页面布局、样式、响应式设计
2. **储值方案展示** - 四个储值方案（500/1200/7000/15000 G Coin）
3. **支付方式说明** - 支持的支付方式（信用卡、Apple Pay等）
4. **安全保障信息** - SSL 加密、绿界验证标识
5. **賣家資訊** - 显示在绿界注册的卖家信息

## 🔧 审核通过后的操作

审核通过后，更新环境变量：

```env
ECPAY_ENABLED=true
ECPAY_STATUS=approved
```

然后：
1. 重启服务
2. 验证页面显示正常
3. 测试创建订单流程

## 🧪 本地预览（可选）

如果你想在本地预览审核期间的页面效果：

1. 配置环境变量：
```env
ECPAY_ENABLED=false
ECPAY_STATUS=pending
```

2. 启动本地服务：
```bash
npm run dev
```

3. 访问：`http://localhost:3001/topup`
